# Backlog — Features de grammaire manquantes (MySQL / MariaDB / Oracle)

## Context

Squishy parse MySQL/MariaDB/Oracle avec des parsers hand-rolled (pas d'ANTLR runtime). Les grammaires `.g4` sous `internal/dialects/*/reference/` servent de spec. Ce document compare ce que les parsers couvrent **aujourd'hui** avec les grammaires de référence, et liste ce qui manque pour pouvoir l'ajouter au fil de l'eau.

Périmètre : DDL principalement (le but de squishy est la migration de schémas + données), avec les éléments PL/SQL/PSM nécessaires pour traduire les routines.

Critères de tri (pour chaque item) :
- **P0** = bloque des migrations réelles courantes
- **P1** = features fréquentes en prod, perte sémantique notable
- **P2** = features rares ou sans équivalent PG (peut être noopé avec warning)

Fichiers de référence :
- MySQL/MariaDB : [MySqlParser.g4](internal/dialects/mysql/reference/MySqlParser.g4), [MySqlLexer.g4](internal/dialects/mysql/reference/MySqlLexer.g4)
- Oracle : [PlSqlParser.g4](internal/dialects/oracle/reference/PlSqlParser.g4), [PlSqlLexer.g4](internal/dialects/oracle/reference/PlSqlLexer.g4)
- Parsers actuels : [mysql/parser.go](internal/dialects/mysql/parser.go), [oracle/parser.go](internal/dialects/oracle/parser.go), [oracle/parser_ddl.go](internal/dialects/oracle/parser_ddl.go), [oracle/parser_plsql.go](internal/dialects/oracle/parser_plsql.go)

> Note : il n'y a **pas** de répertoire `internal/dialects/mariadb/` séparé — MariaDB réutilise le parser MySQL. Les divergences MariaDB sont traitées comme des extensions à brancher dans le même parser.

## Translator gaps transverses

Indépendamment du parsing, ces nœuds AST existent déjà mais sont silencieusement droppés :

- **MySQL/MariaDB** : `Partitions` champ jamais lu par [translator.go](internal/translate/translator.go) (cf. block 206–362), CHECK constraints émis en `rawExpr` sans rewrite (subqueries ou fonctions Oracle/MySQL spécifiques cassent côté PG), FULLTEXT indexes émis en TODO.
- **Oracle** : `ConstraintState` non lu, `USING INDEX` inline ignoré, sequence options Oracle-only droppées sans warning, `DEFAULT ON NULL` ignoré.
- **Generic** : pas de validation post-traduction (noms de constraints uniques, références de colonnes valides, FK cycliques).

## Comment exploiter ce backlog

1. Chaque ligne peut devenir une issue séparée, avec :
   - le numéro (`M-XX` / `MD-XX` / `O-XX`) en référence
   - la règle `.g4` à implémenter
   - la stratégie de traduction PG (ou warning explicite)
   - un dump de test minimal (à ajouter sous `test/integration/`)

2. **Ordre suggéré** :
   1. P0 dans cet ordre : M-01, M-02, O-08, O-09, O-01..O-07, M-03, M-04
   2. P1 par dialecte selon les dumps réels rencontrés
   3. P2 au coup par coup, au minimum noop + warning pour ne pas faire échouer un parse

3. **Conventions** (rappel `internal/dialects/README.md`) :
   - Étendre la grammaire vendored si on remonte des correctifs
   - Ajouter le keyword côté HTTP enum + dropdown Vue si nouvelle "kind"
   - Tests unitaires obligatoires sous `internal/dialects/<dialect>/parser_*_test.go` + intégration sous `test/integration/`

## Vérification

Ce document est un backlog, pas du code. Pour vérifier qu'il reste fidèle à l'état du parser :

```bash
# Re-scanner les TODO/UNSUPPORTED dans le code de dialecte
docker compose --profile test run --rm unit-tests \
  sh -c 'grep -rn "TODO\|FIXME\|UNSUPPORTED" internal/dialects/ internal/translate/'

# Confirmer les top-level rules de chaque .g4
grep -nE "^[a-z_][a-zA-Z_0-9]*\s*:" internal/dialects/mysql/reference/MySqlParser.g4 | head -200
grep -nE "^[a-z_][a-zA-Z_0-9]*\s*:" internal/dialects/oracle/reference/PlSqlParser.g4 | head -200

# Quand une feature de la liste est implémentée :
docker compose --profile test run --rm unit-tests go test ./internal/dialects/<d>/... -run <Test> -v
make e2e   # validation end-to-end avec dumps réels
```

Quand un item passe en "implémenté", le retirer de ce document (ou le déplacer en section "Done" pour traçabilité).

## Done

- ✅ M-01 — MySQL/MariaDB PARTITION BY parsing + lift to PG declarative partitioning (HASH/KEY/RANGE/LIST + COLUMNS variants; SUBPARTITION dropped with note; expression keys downgrade to unpartitioned with warning) (uncommitted)
- ✅ M-02 — MySQL ALTER TABLE: parse MODIFY/CHANGE/RENAME COLUMN, RENAME TO, ALTER COLUMN SET/DROP DEFAULT, ADD INDEX, DROP PK/FK/INDEX, CONVERT TO CHARSET, ALGORITHM/LOCK/FORCE/VALIDATION, DISABLE/ENABLE KEYS, DISCARD/IMPORT TABLESPACE, ORDER BY, alter_partition_specification (consumed as NOOP). New AlterAction kinds wired through AST. (uncommitted)
- ✅ M-03 — MySQL CREATE TABLE LIKE / CREATE TABLE AS SELECT parsing + translation. LIKE → PG `CREATE TABLE … (LIKE src INCLUDING ALL)` pre-action. AS SELECT → blocking manual-review prerequisite preserving the source body. New CreateTableLike / CreateTableAs AST nodes; prereqs builder pre-seeds existing prereqs to avoid wiping per-statement entries. (uncommitted)
- ✅ M-04 — MySQL DROP {INDEX|VIEW|PROCEDURE|FUNCTION|TRIGGER|EVENT}: structured AST `DropObject`, parsed and translated to PG pre-actions. INDEX strips the `ON table` qualifier (PG indexes are schema-namespaced). TRIGGER emits a TODO comment + warning since PG requires the table name. EVENT has no PG equivalent — surfaced as warning only. (uncommitted)
- ✅ M-05 — UNSIGNED integer columns now emit `CHECK (col >= 0)` to mirror MySQL's domain enforcement (TINYINT/SMALLINT/MEDIUMINT/INT/BIGINT UNSIGNED). ZEROFILL is recognised as a display-only attribute and surfaced as a TypeMapping note. (uncommitted)
- ✅ M-06 — `ON UPDATE CURRENT_TIMESTAMP` was already wired in `emitOnUpdateTrigger`; backlog claim was outdated. Added regression tests that lock in the BEFORE UPDATE row-trigger emission for single + multi-column cases. (uncommitted)
- ✅ M-07 — FULLTEXT indexes now emit a real PG GIN expression index over `to_tsvector('simple', coalesce(col, '') || ' ' || …)` instead of a TODO comment. Multi-column indexes concatenate NULL-safely. Surfaced as an info-level Explanation rather than a blocking warning. (uncommitted)
- ✅ M-08 — SPATIAL columns / SPATIAL INDEX were already wired to map to `geometry(SUBTYPE)` + GIST when PostGIS is on the target, with a TEXT fallback + blocking install prereq otherwise. Backlog claim was outdated. Added regression tests for both extension-present and extension-absent paths. (uncommitted)
- ✅ M-09 — Index INVISIBLE/VISIBLE, per-column ASC/DESC, prefix length, and functional `(expr)` keys. New `IndexedCol.Expr/IsExpr` AST fields. PGIndex/pgast.CreateIndex now carry parallel `ColumnDirs`/`ColumnIsExpr` slices; writer emits `(expr)` for expression keys and `DESC`/`ASC` per column. INVISIBLE → info Explanation, prefix-length → warning advising a `LEFT(col, N)` functional index. (uncommitted)
- ✅ M-10 — Advanced table options. Added explicit handlers for `DATA DIRECTORY` / `INDEX DIRECTORY`, `TABLESPACE [STORAGE …]`, `UNION = (…)`. Single-keyword options (STATS_*, COMPRESSION, ENCRYPTION, AVG_ROW_LENGTH, INSERT_METHOD, PASSWORD, DELAY_KEY_WRITE, PACK_KEYS, SECONDARY_ENGINE, CONNECTION, …) flow through the existing K=V catch-all.
